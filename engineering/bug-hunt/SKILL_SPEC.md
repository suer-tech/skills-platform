# Skill Specification

## Name

engineering.bug-hunt

## Purpose

Найти в проекте баги, которые влияют на работоспособность:
логические ошибки, падения, необработанные ошибки,
проблемы с типами, гонки, утечки и т.д.

## Philosophy

Скилл ищет только реальные баги, не code style.
Каждый баг должен воспроизводиться или быть очевидным
по логике кода. Без ложных срабатываний.

## Workflow

1. Scan — прочитать проект, составить карту зависимостей и потоков данных.
2. Detect — найти потенциальные проблемы:
   - null/undefined при обращении
   - отсутствие обработки ошибок
   - логические ошибки в условиях
   - несоответствие типов
   - race conditions
   - мёртвый код с побочными эффектами
   - неправильная работа с ресурсами (файлы, соединения)
3. Verify — проверить каждый кандидат: это действительно баг?
4. Report — сформировать отчёт с приоритетами.
5. Fix (опционально) — если fix=true, исправить баги.

## Success Criteria

- найденные баги — реальные, а не гипотетические;
- для каждого бага указан файл, строка и причина;
- если fix=true — код исправлен и не сломал тесты.
